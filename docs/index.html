MilliHash API (v1.0.0)

Download OpenAPI specification:

Product MilliHash REST API — all product-owned controllers documented per cross-cutting/api-contracts (includes operator admin; excludes Actuator/springdoc).

MilliHash runs as two deployments against the same database:

  • Free instance — directly reachable at api.millihash.com. Serves unauthenticated public verification routes. RapidAPI-authenticated routes (POST/GET /registrations, certificate download) are disabled.
  • Pro instance — reachable only through the RapidAPI proxy (IP whitelist). Serves RapidAPI-authenticated registration and owner-portfolio routes. Public verification routes are not exposed on this deployment.

Authenticated endpoints require X-RapidAPI-Key (pro instance / RapidAPI ingress only).

Public verification endpoints require no authentication (free instance only).

Operator admin (GET /admin/audit) is documented below for completeness. It is served on app.millihash.com with Auth0 JWT — not via RapidAPI or the public api.millihash.com marketplace edge.

Registrations

Asset hash registration (pro / RapidAPI only)

Register an asset hash with RFC 3161 TSA timestamp

Synchronous registration pipeline: resolve or accept SHA-256 hash as documentHash, stamp contentHash (equal at standalone genesis), acquire TSA token on contentHash, enforce global documentHash exclusion, persist Registration, enqueue AnchorJob. Asset bytes are never stored. Client-supplied mode accepts hash hex only (no file upload).
